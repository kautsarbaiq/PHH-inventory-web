# PHH Inventory — Backend Deployment Guide (Render.com)

This document provides step-by-step instructions to deploy the **PHH Inventory Backend Server** (Node.js/Express pnpm monorepo) on **Render.com**. 

---

## 📋 Prerequisites
Before starting, ensure you have:
1. **GitHub Access:** Read/Write access to the repository: `https://github.com/kautsarbaiq/PHH-inventory-web.git`
2. **PostgreSQL Database:** An active PostgreSQL connection string (e.g., from Supabase or Neon.tech).

---

## 🚀 Step-by-Step Deployment Instructions

### 1. Create a New Web Service
1. Log in to [Render Dashboard](https://dashboard.render.com/) using your GitHub account.
2. Click the **New +** button in the top right corner and select **Web Service**.
3. Under **Connect a repository**, select your repository: **`PHH-inventory-web`**.

---

### 2. Configure Service Settings
Fill in the deployment configuration fields exactly as detailed below:

| Field | Configuration Value |
| :--- | :--- |
| **Name** | `phh-inventory-backend` *(or any preferred name)* |
| **Region** | `Singapore (ap-southeast-1)` *(recommended for Indonesia/Southeast Asia region speed)* |
| **Branch** | `main` |
| **Root Directory** | *Leave blank* |
| **Runtime** | `Node` |
| **Build Command** | `npm install -g pnpm && pnpm install --frozen-lockfile && pnpm -r build` |
| **Start Command** | `pnpm --filter @phh/server start` |
| **Instance Type** | `Free` *(or upgraded Tier based on production requirements)* |

---

### 3. Configure Environment Variables
You must set the runtime environment variables for the application to establish database connections and handle authentication.

1. Scroll down and click the **Advanced** button (or navigate to the **Environment** tab on the left sidebar after creating the service).
2. Click **Add Environment Variable** and enter the following key-value pairs:

| Key | Value | Description |
| :--- | :--- | :--- |
| **`DATABASE_URL`** | `postgresql://postgres.txwsocepzzvnghjxasjj:Kautsar_123@aws-1-ap-northeast-1.pooler.supabase.com:5432/postgres` | *Your active PostgreSQL database connection string (use this verified regional connection pooler URL to bypass Render IPv6 routing limits).* |
| **`BETTER_AUTH_SECRET`** | `phh-inventory-secret-key-must-be-32-chars-long` | *A secure, random 32-character string for authentication hashing.* |
| **`PORT`** | `10000` | *The default port allocated by Render.* |
| **`CLIENT_URL`** | `https://your-frontend-domain.vercel.app` | *The secure URL of the deployed Vercel frontend (can temporarily use `http://localhost:5173` until frontend deployment is finalized).* |

3. Click **Save Changes** or click **Create Web Service** at the bottom of the page.

---

### 4. Deploy and Verify
* Render will automatically pull the codebase, install dependencies, trigger the workspace builds, and launch the server.
* Once the build completes, the status indicator will turn **Live** (Green).
* You can verify the API is active by accessing the secure URL generated by Render (e.g., `https://phh-inventory-backend.onrender.com/health` or `/`).
